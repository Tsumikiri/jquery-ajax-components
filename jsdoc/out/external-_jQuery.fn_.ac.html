Namespace: ac

"jQuery.fn". ac

This namespace includes just two items: the main plugin function (detailed below) and the config sub-namespace, which holds configurable plugin settings.

Methods

<static> $.fn.ac(eventName, filterSelector, ajaxOptions, alertOptions)

The plugin function.

This is the main function for this plugin. It binds a special event handler to the given page element(s). This handler makes an ajax request which must return a JSON response. That response can leverage several special keys ("content" is the most common) to manipulate the page. Read more about this in the Server Response tutorial.

The name of an event to use for the binding, such as 'click' or 'submit'

Returns:

A pseudo-deffered object (like a promise but dumber) that will be resolved or rejected with any ajax call made by the component(s)

Type
PseudoDeferred
Examples

Initializing an ajax link. Alternatively, you can leave out the second argument and instead use a data attribute on the anchor tag e.g. data-ac-ajax-url="/path/to/remote/function/".

$('#my-link').ac('click', {
    url: '/path/to/remote/function/'
});

Initializing an ajax form. The URL and method will be automatically taken from the form's action and method attributes.

$('#my-form').ac('submit');

Chaining on the pseudo-deferred object returned by the function. Note that unlike a normal promise, the pseudo-deferred object is too dumb to know what status it's in, so it can be resolved and rejected multiple times. As a side-effect it cannot automatically run newly-added callbacks for already completed events.

$('#my-form').ac('submit').fail(function(xhr, status, error) {
    console.log(error);
});

An alternative for using the deferred object by instead using the error parameter of $.ajax.

$('#my-form').ac('submit', {
    error: function(xhr, status, error) {
        console.log(error);
    }
});
