AjaxComponents

This anonymous scope encloses all Ajax Component functionality. Members of this namespace are not accessible outside of the plugin script. This reduces namespace pollution and helps to abstract away the underlying implementation of the plugin.

(private, inner) ajaxWrapper(optsnullable, deferrednullable) → {jqXHR}

The plugin's ajax wrapper

Wrapper for jQuery.ajax with response processing and ajax event handlers included. The function takes the default configuration options (see jQuery.ac.config.ajaxDefaults) and deep-extends them with the given options. Calls AjaxComponents~processResponse on success. Calls jQuery.ac.config.doneHandler, jQuery.ac.config.errorHandler, and jQuery.ac.config.alwaysHandler as appropriate when the ajax response is received. This function is also exposed as jQuery.ac. If deferred is passed, this function will call AjaxComponents~PseudoDeferred#resolve or AjaxComponents~PseudoDeferred#reject as appropriate when the ajax response is received.

(private, inner) alertWrapper(alertOptsopt, nullable, ajaxOptsopt, nullable, deferredopt, nullable)

Wrapper for swal that can default to built-in alerts if swal is not available. The function takes the default configuration options (see jQuery.ac.config.alertDefaults) and deep-extends them with the given options. If swal is not available, the function will use the built in alert, prompt, or confirm functions. Note that these functions halt JavaScript processing while waiting for user input, whereas SweetAlert does not. Also, if using system alerts the type, title, text, and inputValue settings are avaiable for alertOpts. This function calls AjaxComponents~ajaxWrapper when the user confirms the alert. If the user cancels the alert, nothing happens.

(private, inner) componentEventHandler() → {boolean}

Event handler to bind to initialized components. When fired, this function first builds its configuration data, starting with the data attached to the event and deep-merging it with the data attached to the element via data attributes (see AjaxComponents~extendWithData). After building the configuration data, the function checks for the presence of alert options. If alert options are present, it calls AjaxComponents~alertWrapper, else it calls AjaxComponents~ajaxWrapper.

(private, inner) extendWithFormData(datanon-null, elementnon-null)

Adds form data from a page element (form tag) to a given object. If the element is not a form tag, the function exits without doing anything. The form tag's action attribute is mapped to data.ajax.url and its method attribute is mapped to data.ajax.method. The form is also serialized (see jQuery.ac.config.serializeObject), and the serialized data is mapped to data.ajax.data.

Inserts a value into an object, safely extending any sub-objects or arrays. If an object is being inserted into a key that already contains an object, the original object is deep-extended by the new object. If an array is being inserted into a key that already contains an array, the new array is appended to the old one. Otherwise, the value is simply inserted into the key.

(private, inner) processResponse(data)

Processes directives in ajax response data

If response handling is enabled (it is by default; see jQuery.ac.config.enableResponseHandlers), looks for each expected directive key in the data. Then runs the handler for each found key. If the data at the directive key is an object or array, the handler is run as a callback for jQuery.each over the data, else the handler is called directly with the data as the first (and second for compatibility) arguments.
