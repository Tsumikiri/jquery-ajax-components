The plugin's ajax wrapper.

This is a wrapper for $.ajax that includes the response handler functionality of the Ajax Components plugin. Access to this function is provided so that the plugin's response handler functionality can be used outside of an actual component element.

ac(eventNameopt, non-null, filterSelectoropt, nullable, ajaxConfigurationopt, nullable, alertConfigurationopt, nullable) → {AjaxComponents.PseudoDeferred|jQuery}

The plugin function.

This is the main function for this plugin. It binds a special event handler to the given page element(s). This handler makes an ajax request which must return a JSON response. That response can leverage several special keys ("html" is the most common) to manipulate the page. Be sure to check out the getting-started tutorial!

Examples

Initializing an ajax link. Alternatively, you can leave out the second argument and instead use a data attribute on the anchor tag (e.g. data-ac-ajax-url="/path/to/remote/function/").

$('#my-link').ac('click', {
    url: '/path/to/remote/function/'
});

Initializing an ajax form. The URL and method will be automatically taken from the form's action and method attributes.

$('#my-form').ac('submit');

Chaining on the pseudo-deferred object returned by the function. Note that unlike a normal promise, the pseudo-deferred object is too dumb to know what status it's in, so it can be resolved and rejected multiple times. As a side-effect it cannot automatically run newly-added callbacks for already completed events.

$('#my-form').ac('submit').fail(function(xhr, status, error) {
    console.log(error);
});

An alternative for using the deferred object by instead using the error parameter of $.ajax.

$('#my-form').ac('submit', {
    error: function(xhr, status, error) {
        console.log(error);
    }
});

This code snippet runs the ajax call immediately when the component is initialized as well as on click.

$('#my-link').ac('ac.init click', {
   //ajax options...
});
